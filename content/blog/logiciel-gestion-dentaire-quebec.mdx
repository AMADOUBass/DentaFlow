Le marché des logiciels de gestion dentaire est vaste, mais peu d'options sont conçues **spécifiquement pour le Québec**. Voici les critères qui devraient guider votre choix.

## 1. Hébergement canadien (obligation légale)

La Loi 25 impose que les données personnelles de résidents québécois soient hébergées au Canada, ou que des protections équivalentes soient en place pour l'hébergement à l'étranger.

Méfiez-vous des logiciels américains qui hébergent par défaut sur des serveurs aux États-Unis — même avec un accord de confidentialité, cela peut créer des zones grises.

**À chercher :** Hébergement AWS Canada (Montréal), Azure Canada, ou Supabase Canada.

## 2. Support bilingue natif FR/EN

Vos patients sont francophones. Votre logiciel doit l'être aussi, avec des communications automatiques (SMS, emails) en français. Certaines cliniques ont une clientèle anglophone — le support bilingue n'est alors pas un luxe.

## 3. Conformité RAMQ

La gestion des numéros RAMQ doit être traitée avec soin : stockage chiffré, accès tracé, droit de suppression. Vérifiez que le logiciel dispose d'un audit log dédié.

## 4. API ouverte pour intégrations

Votre logiciel de facturation, votre CRM patient, votre agenda Google — ils doivent pouvoir parler à votre logiciel de gestion clinique. Une API REST documentée est aujourd'hui indispensable.

## 5. Prise de rendez-vous en ligne intégrée

Un site web qui renvoie les patients vers Calendly ou une autre solution externe, c'est une couture visible qui dégrade l'expérience. La prise de RDV doit être native, avec calcul automatique des disponibilités par praticien.
